# Использование localStorage в проекте

## Где используется localStorage:

### 1. **Авторизация** (`src/contexts/AuthContext.tsx`)
- **Ключ:** `admin_authenticated`
- **Значение:** `'true'` или отсутствует
- **Назначение:** Сохранение статуса авторизации администратора
- **Когда:** После успешного входа в админ-панель

### 2. **Язык интерфейса** (`src/contexts/LanguageContext.tsx`)
- **Ключ:** `selected_language`
- **Значение:** `'ru'`, `'ro'` или `'en'`
- **Назначение:** Сохранение выбранного языка пользователя
- **Когда:** При каждом изменении языка

### 3. **Портфолио проекты** (`src/lib/db.ts` - fallback)
- **Ключ:** `portfolio_projects`
- **Значение:** JSON массив проектов
- **Назначение:** Fallback хранилище, если Supabase недоступен
- **Когда:** Только при ошибке подключения к Supabase

### 4. **Статьи блога** (`src/lib/db.ts` - fallback)
- **Ключ:** `blog_articles`
- **Значение:** JSON массив статей
- **Назначение:** Fallback хранилище, если Supabase недоступен
- **Когда:** Только при ошибке подключения к Supabase

## Приоритет хранения данных:

1. **Основное хранилище:** Supabase (PostgreSQL)
2. **Fallback:** localStorage (только если Supabase недоступен)

## Безопасность:

⚠️ **Важно:**
- Пароли НЕ хранятся в localStorage
- Только статус авторизации (`admin_authenticated`)
- Данные проектов и статей синхронизируются с Supabase
- localStorage используется только как временный fallback
